## 报告安全漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请通过以下方式报告：

### 报告方式

1. **通过 GitHub 安全建议**（推荐）
   - 访问 https://github.com/smarttoken/smarttoken/security/advisories/new
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **通过电子邮件**
   - 发送邮件至：snhsl@sina.com
   - 邮件主题：[SECURITY] 问题描述

### 报告内容

请包含以下信息：

- 问题类型
- 受影响的组件
- 复现步骤
- 可能的利用场景
- 建议的修复方案（如果有）

### 响应时间

我们承诺：

- **24 小时内**确认收到报告
- **7 天内**提供初步反馈
- **30 天内**发布修复或说明

### 安全更新

安全更新将通过以下渠道发布：

- GitHub Security Advisories
- 新版本发布 (v1.0.1, v1.0.2, ...)
- Release Notes 说明

## 安全最佳实践

使用 SmartToken 时，请遵循以下安全最佳实践：

### API Key 安全

1. **不要在代码中硬编码 API Key**
   - 使用环境变量或配置文件
   - 启用加密存储

   ```bash
   export OPENAI_API_KEY=sk-xxx
   export DEEPSEEK_API_KEY=sk-xxx
   ```

2. **使用强主密码**
   - 至少 12 个字符
   - 包含大小写字母、数字和特殊字符

3. **定期轮换 API Key**
   - 每 90 天更换一次
   - 立即更换如果怀疑泄露

### 网络安全

1. **使用 HTTPS**
   - 生产环境务必使用 HTTPS
   - 启用 TLS 1.2 或更高版本

2. **限制访问**
   - 只允许必要的 IP 访问
   - 使用防火墙规则

3. **监控日志**
   - 定期检查访问日志
   - 设置异常访问告警

### 数据安全

1. **本地存储**
   - 敏感数据存储在本地
   - 使用加密的文件系统

2. **定期备份**
   - 备份配置文件
   - 备份数据库

3. **安全清理**
   - 不使用时锁定 vault
   - 定期清理缓存
